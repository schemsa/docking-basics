# 010 - Calculating redocking RMSD

Today I calculated the RMSD between the crystallographic ligand pose and the best docked pose from AutoDock Vina.

## Files used

- `prep/ligand_E20.pdb`
- `res/E20_docked_pose1.pdbqt`

## Goal

The goal was to evaluate whether the docking protocol could reproduce the original crystallographic ligand position.

## First RMSD attempt

The first calculation gave:

```text
Kabsch RMSD = 6.498 Å

However, this result was not reliable because the atom order differed between the original and docked files.

Example:

original C10 | docked O24

This showed that the atoms were not being compared correctly.

Corrected RMSD calculation

The script was updated to match atoms by atom name instead of relying on file order.

Final result
Original atoms: 28
Docked atoms: 28
Common atoms: 28

Centroid distance: 0.453 Å

Raw RMSD = 0.768 Å
Kabsch RMSD = 0.572 Å
Interpretation

The redocking result was successful.

A redocking RMSD below 2 Å is generally considered good for reproducing a crystallographic binding pose.

The final Kabsch RMSD of 0.572 Å indicates that the docked pose closely matches the original ligand position.

Important lesson

For ligand RMSD calculations, atom mapping matters.

If atom order changes during file conversion, RMSD calculations based only on atom order can give misleading results.

In this case, atom-name matching produced a reliable comparison.
